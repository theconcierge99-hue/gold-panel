Overview & pricing

Executive Lounge is a private intelligence terminal: live market wire, RWA-tokenized creator signals (optional Solana NFT), and paid Concierge AI. The UI is English; Concierge mirrors the user’s language when clear from their messages.

ActionPrice
Browse headlinesFree
Open RSS article0.1 USDC
Concierge message0.1 USDC
Unlock creator signal0.1 USDC
Publish creator signal1 USDC

Reader unlock fees split 50% creator / 50% merchant. Publish fee is 100% merchant. Solana publish also requires ~0.03 SOL in Phantom for NFT mint gas (after 1 USDC).

Architecture

  • UI — executive-lounge.html + x402-pay.mjs client bundle
  • APIs — Vercel Edge functions under /api/*
  • AI — Google Gemini
  • Payments — x402 v2 via PayAI facilitator (on-chain settlement)
  • Trust — optional zauth.inc directory + Provider Hub telemetry
  • Storage — Redis/KV for signals, RWA certificates, badges, ledger, Concierge memory
  • RWA mint — mint-signal.mjs + /api/solana-rpc-send (Phantom, Metaplex)

API reference

Free

  • GET /api/market — headline feed + ticks
  • GET /api/x402-config — public payment config (zauthTelemetryEnabled, discovery.zauth links)
  • GET /api/zauth-directory — proxy to zauth x402 endpoint directory (CORS for agents)
  • GET /api/zauth-status — this site’s directory slice + verification checks
  • GET /.well-known/x402 — x402 discovery fan-out
  • GET /openapi.json — OpenAPI + payment metadata
  • GET /.well-known/agent-card.json — agent registry / how to register agt_…
  • POST /api/agent-identity — register agent (public keys)
  • GET /api/agent-identity — profile, directory (?list=1), or card
  • GET /api/agent-identity-card — ERC-8004-style card per agent

Paid (POST, x402)

  • POST /api/news-open — unlock article URL
  • POST /api/concierge — AI chat / enhance / image
  • POST /api/lounge-signal-publish — publish signal + RWA cert (1 USDC)
  • POST /api/lounge-signal-open — unlock signal body (0.1 USDC)

RWA (free reads)

  • GET /api/rwa-token?signalId= — certificate
  • GET /api/rwa-metadata?signalId= — NFT JSON metadata
  • GET /api/rwa-badges?wallet= — reader badges
  • POST /api/lounge-rwa-record-mint — save Phantom mint result
  • POST /api/solana-rpc-send — Solana RPC proxy for browser mint

Without payment, paid routes return 402 with PAYMENT-REQUIRED. GET on paid routes also returns 402 for marketplace probes.

x402 payments

USDC on Base and optionally Solana. Facilitator: PayAI (facilitator.payai.network).

  1. Client receives 402 + payment requirements
  2. Wallet signs USDC transfer
  3. Client retries with PAYMENT-SIGNATURE
  4. Server verifies and settles; response includes PAYMENT-RESPONSE with transaction hash

Set X402_EVM_PAY_TO and/or X402_SOL_PAY_TO in your host environment. Never commit keys or RPC URLs with embedded secrets.

zauth integration

zauth is the security and trust layer for paid x402 APIs — endpoint directory, uptime, verification badges, and optional Provider Hub telemetry. It does not replace PayAI settlement or wallet signing; payments still use the normal x402 client (x402-pay.mjs).

What Lounge exposes

Route / fieldPurpose
GET /api/zauth-directoryProxy to zauth Database — agents query before paying third-party APIs
GET /api/zauth-statusEndpoints for this deployment + live verification when configured
discovery.zauth in /api/x402-configHub, database, and docs URLs

Provider Hub (operators)

  1. Create an API key at zauth.inc/provider-hub
  2. Set ZAUTH_API_KEY in Vercel (never commit it)
  3. Redeploy; complete real paid traffic on Concierge, news-open, signal routes
  4. Check GET /api/zauth-status — providerTelemetryEnabled: true

After each successful paid 200, the server sends async telemetry (request, response, payment) to zauth. Failures are logged only; user responses are unchanged.

Paid routes that report

  • POST /api/concierge
  • POST /api/news-open
  • POST /api/lounge-signal-publish
  • POST /api/lounge-signal-open

Agents & Concierge

Before recommending external x402 APIs, agents should prefer verified endpoints with high successRate:

GET /api/zauth-directory?verified=true&search=example.com&limit=20

Concierge is instructed to use this directory when suggesting third-party paid services.

Concierge AI

Modes: chat, enhance (signal editor), image (analysis + optional visual).

Uses live market data, general-knowledge feeds, lounge memory (recent headlines + creator signals), and an internal DeFi desk block (TVL, yields, whales, wallet snapshot, verdict) when the question is on-chain / DeFi.

Language: defaults to English; replies in Indonesian or other languages when the user’s message (or recent user turns for short replies) is clearly in that language.

External agents: integrate via x402 without source access — see Agents & integrators (POST /api/concierge, 0.1 USDC, OpenAPI + Node/Python examples).

Concierge Intel APIs (integrators): separate paid endpoints per function — TVL · Yields · Whales · Wallet · Verdict at 0.1 USDC each (POST /api/concierge-intel-*).

Agent identity: register agt_… + Solana/Base wallets for A2A discovery — see Agent identity (Lounge → Navigate → Agent identity, or POST /api/agent-identity). Optional header X-Agent-Id on paid Concierge calls.

RWA (Real World Assets)

Each published signal is an RWA intelligence certificate (off-chain tokenId + content hash). Solana creators mint a Metaplex NFT to their own Phantom wallet after paying 1 USDC — gas in SOL (~0.03).

Mint flow: publish → approve USDC → approve NFT in Phantom → POST /api/lounge-rwa-record-mint. RPC for mint uses /api/solana-rpc-send (set SOLANA_RPC_URL to Helius on Vercel).

Readers earn off-chain badges (Intel Scout → Sovereign Intel) when unlocking signals. On-chain name is limited to 32 UTF-8 bytes — shorten long titles if mint fails with error 0xb.

Full operator guide: repository docs/rwa.md and docs/rwa-solana-setup.md.

Creator signals

Mint & Publish stores a signal + RWA cert in KV and surfaces it on the market feed (⬡ RWA chip). Unlock shows the full summary in a modal.

Requires KV_REST_API_URL and KV_REST_API_TOKEN (or Upstash equivalents) in production.

Security

  • Never commit API keys, KV tokens, private keys, or RPC URLs with keys
  • Do not paste full /api/x402-config responses in public support threads
  • Restrict Gemini keys by HTTP referrer in Google AI Studio
  • Rotate any credential that was exposed

Extended markdown docs for operators live in the repository docs/ folder (same content, security-reviewed).

Deployment

Deploy via Git → Vercel on the correct team and project. Avoid vercel deploy from a personal account that lacks production env vars.

After deploy, verify: site loads, /deploy-version.txt matches your commit, /api/x402-config shows enabled: true, and paid routes return 402 without payment.
