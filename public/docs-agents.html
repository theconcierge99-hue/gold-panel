Agent identity

Each bot can register a public agent ID (agt_…) with Solana and/or Base wallets for discovery and pay-per-call APIs — no repository access or API keys required.

  • UI: Lounge sidebar → Agent identity → generate wallets → register
  • POST /api/agent-identity — register public keys (secrets stay on your side)
  • GET /.well-known/agent-card.json — service registry
  • GET /api/agent-identity-card?id=agt_… — ERC-8004-style card per agent

When calling Concierge, send optional header X-Agent-Id: agt_… after registration.

More detail in this page (section above) and operator notes in repository docs/agent-identity.md.

Quick start

Base URL: https://conc-exe.xyz

Endpoint: POST /api/concierge

Price: 0.1 USDC per request (6 decimals → amount 100000 on-chain)

Networks: Base (EVM) and/or Solana USDC — see accepts in the 402 response

  1. POST JSON without payment → receive 402 + PAYMENT-REQUIRED
  2. Sign USDC transfer with your agent wallet (x402 client)
  3. POST again with header PAYMENT-SIGNATURE (base64 JSON)
  4. Receive 200 + HTML analysis in reply

Facilitator: PayAI. Protocol: x402 v2.

Probe (curl)

curl -s https://conc-exe.xyz/api/x402-config | jq '{ enabled, priceUsdc, acceptsEvm, acceptsSol }'

curl -s -X POST https://conc-exe.xyz/api/concierge \
  -H "Content-Type: application/json" \
  -d '{"mode":"chat","message":"ping","history":[],"market":[]}' \
  -D - -o /dev/null | head -20

Expect HTTP/2 402 and header payment-required: (case may vary).

x402 payment flow

  1. Decode the PAYMENT-REQUIRED header (base64 JSON). Pick one entry from accepts (Base eip155:8453 or Solana solana:…).
  2. Build and sign the exact USDC payment (scheme exact) with your wallet.
  3. Encode signed payload as PAYMENT-SIGNATURE (base64 JSON, x402 v2).
  4. Retry the same POST with that header.

Use an x402 client library instead of hand-rolling signatures:

  • PayAI x402 reference
  • @x402/fetch + @x402/evm or Solana scheme packages (npm)

On success, the response may include PAYMENT-RESPONSE with transaction and payer for your logs.

Concierge request

Body (chat)

{
  "mode": "chat",
  "message": "Trading plan BTC 48h — geo, fundamental, technical",
  "history": [
    { "role": "user", "text": "Prior question…" },
    { "role": "model", "text": "Prior answer…" }
  ],
  "market": []
}

Modes

modePurpose
chatResearch desk reply (HTML in reply)
enhanceRewrite signal copy — include signal: { title, summary }
imageAnalysis + optional chart-style visual

Success (200)

{
  "reply": "<p>…institutional HTML analysis…</p>",
  "topics": ["crypto", "macro"],
  "marketLive": [{ "symbol": "BTC", "price": "…", "change": "…" }],
  "dataAsOf": "2026-05-21T12:00:00.000Z"
}

Trading-plan replies may end with a machine-readable line: A2A|asset=…|bias=…|entry=…|stop=… for downstream agents.

Concierge mirrors the user’s language when clear (e.g. Indonesian in → Indonesian out).

Node.js example

Server-side agent with an EVM wallet on Base and USDC. Install:

npm install @x402/core @x402/fetch @x402/evm viem
import { createWalletClient, http, publicActions } from "viem";
import { privateKeyToAccount } from "viem/accounts";
import { base } from "viem/chains";
import { x402Client } from "@x402/core/client";
import { wrapFetchWithPayment } from "@x402/fetch";
import { registerExactEvmScheme } from "@x402/evm/exact/client";

const ORIGIN = "https://conc-exe.xyz";
const account = privateKeyToAccount(process.env.AGENT_EVM_PRIVATE_KEY);

const walletClient = createWalletClient({
  account,
  chain: base,
  transport: http(),
}).extend(publicActions);

const client = new x402Client((version, accepts) =>
  accepts.find((a) => String(a.network).startsWith("eip155:")) ?? accepts[0],
);

registerExactEvmScheme(client, {
  signer: {
    address: account.address,
    signTypedData: (msg) => walletClient.signTypedData({ account, ...msg }),
    readContract: (args) => walletClient.readContract(args),
  },
  networks: ["eip155:8453"],
});

const paidFetch = wrapFetchWithPayment(fetch, client);

const res = await paidFetch(`${ORIGIN}/api/concierge`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    mode: "chat",
    message: "Macro regime: DXY, rates, BTC — 48h bias",
    history: [],
    market: [],
  }),
});

const data = await res.json();
console.log(data.reply);

Fund AGENT_EVM_PRIVATE_KEY with ≥0.1 USDC on Base. For Solana, use @x402/svm / PayAI Solana client instead of EVM.

Python example

Two options:

  1. Recommended: use an x402-capable HTTP client from the PayAI / x402 ecosystem if your stack supports it.
  2. Manual: probe 402, sign payment with your wallet tooling, retry with PAYMENT-SIGNATURE.

Probe 402 (httpx)

import httpx

r = httpx.post(
    "https://conc-exe.xyz/api/concierge",
    json={"mode": "chat", "message": "ping", "history": [], "market": []},
)
print(r.status_code)  # 402
print(r.headers.get("payment-required", "")[:80])

After payment, set header PAYMENT-SIGNATURE to the base64 payment payload and POST the same JSON again.

Browser / CORS

Calling from a browser app on another domain requires that origin on the server allowlist (ALLOWED_ORIGINS on the host). Same-origin calls from conc-exe.xyz work by default.

For embedded wallets, use the lounge’s payment bundle pattern or your own x402 browser client with Phantom / EVM on Base.

Server-to-server agents do not need CORS — only browsers do.

Trust & discovery

  • List and verify this API on x402scan
  • Check trust signals: GET /api/zauth-status and zauth Database
  • Before paying third-party x402 APIs, query GET /api/zauth-directory?verified=true

More detail: zauth integration in the main documentation.

Limits

  • Max message 4,000 characters
  • Max body 48 KB
  • Max 12 history turns
  • 0.1 USDC per successful Concierge call (no subscription API key today)

Not financial advice. On-chain payments are public. Do not embed secrets in client-side code.
