Platform guard

Hard-blocked targets (403 platform_scope_forbidden):

  • conc-exe.xyz and subdomains
  • Deployment hosts (VERCEL_URL, X402_SITE_ORIGIN, …)
  • Private networks, IP literals, punycode, non-standard ports
  • Redirect hops to any forbidden host

Paid routes require allowlist + authorized: true. Rate limits: 20/min (scope), 8/min (audits) per IP.

Scope validation

bash / macOS / Linux

curl -s -X POST https://conc-exe.xyz/api/concierge-security-scope \
  -H "Content-Type: application/json" \
  -d '{"target":"https://api.example.com","allowlist":["*.example.com"]}'

Windows PowerShell — use curl.exe (one line) or:

Invoke-RestMethod -Method POST -Uri "https://conc-exe.xyz/api/concierge-security-scope" -ContentType "application/json" -Body '{"target":"https://api.example.com","allowlist":["*.example.com"]}'

Test denylist: use your-preview.vercel.app as target. 403 = platform blocked · 200 = external allowlist pass.
